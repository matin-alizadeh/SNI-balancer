# SNI-Balancer

یک بالانسر هوشمند برای کانفیگ‌های Xray در محیط‌های مبتنی بر SNI Spoofing

این پروژه به‌صورت خودکار کانفیگ‌های VLESS و Trojan را با تست واقعی سرعت دانلود و latency بررسی می‌کند، بر اساس عملکرد و پایداری به آن‌ها امتیاز می‌دهد و بهترین کانفیگ را به‌عنوان یک پراکسی SOCKS5 از طریق Xray اجرا نگه می‌دارد.

مناسب برای شبکه‌های دارای محدودیت و فیلترینگ که نیاز به SNI Spoofing دارند.

بدون نیاز به v2rayN یا کلاینت‌های GUI
کاملاً خودکار

* پشتیبانی از Subscription URL
* پشتیبانی از REALITY
* پشتیبانی از TLS
* چندسکویی:

  * لینوکس

  * macOS

# نحوه عملکرد

1. خواندن کانفیگ‌ها از `configs.txt`
2. ساخت نمونه‌های موقت Xray برای تست
3. انجام:

   * Health Check
   * تست Latency
   * تست واقعی سرعت دانلود
4. محاسبه امتیاز هر کانفیگ بر اساس:

   * سرعت
   * پینگ
   * پایداری
5. اجرای بهترین کانفیگ
6. تست مجدد در بازه‌های زمانی مشخص
7. سوییچ فقط در صورت بهتر بودن محسوس کانفیگ جدید
8. ذخیره تاریخچه تست‌ها

# نصب

## کلون کردن پروژه

```bash
git clone https://github.com/yourname/sni-balancer.git
cd sni-balancer
```

## نصب وابستگی‌ها

```bash
pip install -r requirements.txt
```

## ساخت فایل `configs.txt`

پشتیبانی از:

* `vless://`
* `trojan://`
* Subscription URL
* Subscription Base64

مثال:

```text
vless://uuid@host:port?...#MyConfig
trojan://password@host:port?...#Server2
https://subscription-url.example/sub
```

---

# اجرا

## اجرای عادی

```bash
python3 balancer.py
```

## حالت تست بدون اجرای Xray

```bash
python3 balancer.py --dry-run
```

## تغییر بازه تست

```bash
python3 balancer.py --interval 300
```

## تغییر پورت SOCKS5

```bash
python3 balancer.py --port 1080
```

## تست سرعت بزرگ‌تر

```bash
python3 balancer.py --test-size 10
```

## آپدیت Xray

```bash
python3 balancer.py --update-xray
```

# آرگومان‌ها

| آرگومان          | توضیح                  |
| ---------------- | ---------------------- |
| `--dry-run`      | فقط تست کانفیگ‌ها      |
| `--interval`     | فاصله زمانی تست‌ها     |
| `--configs`      | مسیر فایل کانفیگ       |
| `--port`         | پورت SOCKS5            |
| `--display-time` | مدت نمایش کامل داشبورد |
| `--test-size`    | حجم تست دانلود         |
| `--update-xray`  | دانلود/آپدیت Xray      |
| `--sni-variant`  | انتخاب Rust یا Go      |
| `--sni-connect`  | آدرس upstream          |
| `--sni-fake`     | Fake SNI               |

# سیستم امتیازدهی

هر کانفیگ بر اساس سه معیار امتیاز می‌گیرد:

| معیار   | وزن |
| --- | ------- |
| سرعت    | ۴۰٪ |
| پایداری | ۳۰٪ |
| پینگ    | ۳۰٪ |
برای جلوگیری از سوییچ‌های غیرضروری، فقط زمانی تغییر انجام می‌شود که اختلاف امتیاز از مقدار مشخصی بیشتر باشد.

کانفیگ‌هایی که چند بار پشت‌سرهم fail شوند، موقتاً کنار گذاشته می‌شوند.

* پراکسی SOCKS5 به‌صورت پیش‌فرض روی تمام اینترفیس‌ها فعال است

* در صورت کرش Xray، برنامه به‌صورت خودکار آن را مجدد اجرا می‌کند
* نام‌های تکراری کانفیگ‌ها به‌صورت خودکار یکتا می‌شوند
